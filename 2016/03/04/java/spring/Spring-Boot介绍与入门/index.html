Spring Boot介绍与入门

Mar 4, 2016 | JavaSpring | 阅读

Spring Boot的介绍可以看这篇文章。总得来说，就是Spring也觉得自己用起来和什么PHP web框架啊，ROR框架啊，Python web框架啊相比太麻烦了，自己也不好意思了，现在大伙又喜欢推崇什么’微框架’(参考文章)，讲究的就是灵活，简洁，易测试，易部署。所以Spring作为Java框架的老大，就祭出了Spring Boot这么一个东西。在Spring强大的基础上，弄了很多简化配置，部署的功能。

以下主要参考Spring Boot的官网文档。

安装

我使用sdkman来安装Spring boot。sdkman的教程请看SDKMAN的安装与使用 | 木杉的博客。

Hello World

spring boot宣传的时候，重点渲染的就是他比传统Spring能更快的搭建起来，更快的出原型，那到底有多快？我们做个hello world试试。

把上面的代码写入app.groovy，然后执行：

spring run app.groovy

第一次执行的时候，Spring boot需要下载依赖，所以会比较慢。之后启动就不会了。

启动成功后浏览器中打开http://localhost:8080/，就可以看到Hello World!啦。

虽然一个文件就可以弄个demo对于PHP，Python之流是家常便饭，但是对于Java，对于Spring，还是一件令人兴奋事情的。。。

运行应用

因为使用maven管理项目，所以也使用maven运行项目。

$ mvn spring-boot:run

启动后浏览器访问http://localhost:8080/就看到Hello World!啦！

问题

Spring可以做到热更新代码么?

保存文件后，刷新网页就可以看到效果，这个是PHP之流说Java之流傻逼的铁证。个人觉得也是PHP在web开发上，使用比例远比Java高的原因之一。谁不想保存代码就能看到效果呢。

Spring boot也考虑到了这一点，推出了spring-boot-devtools。他会监视classpath，一旦发现有类文件更新了，就自动重启。
